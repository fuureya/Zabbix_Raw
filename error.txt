🚨 MASALAH BARU: DATABASE VERSION MISMATCH DI SERVER

MASALAH:
Database version: 7040000 (Zabbix 7.4)
Container version: 7000000 (Zabbix 7.0.8)
Error: "The Zabbix database version does not match current requirements"

📋 SOLUSI TERSEDIA:

🎯 SOLUSI 1: UPGRADE KE ZABBIX 7.4 (RECOMMENDED)
1. Gunakan file: docker-compose-server.yml
   docker-compose -f docker-compose-server.yml up -d

2. Atau update docker-compose.yml yang sudah dimodifikasi:
   - zabbix/zabbix-server-mysql:7.4-ubuntu-latest
   - zabbix/zabbix-web-nginx-mysql:7.4-ubuntu-latest

⚠️ SOLUSI 2: DOWNGRADE KE ZABBIX 7.0 (DATA LOSS RISK)
1. Otomatis: ./migrate-to-zabbix70.sh
2. Manual: Backup → Reset DB → Start fresh

🔧 TOOLS YANG SUDAH DIBUAT:
✅ docker-compose-server.yml (untuk Zabbix 7.4)
✅ migrate-to-zabbix70.sh (downgrade script)
✅ check-compatibility.sh (version checker)
✅ README-SERVER-MIGRATION.md (panduan lengkap)
